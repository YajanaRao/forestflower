1. Forest Flower                            *forestflower.nvim-forest-flower*

A Neovim colorscheme combining warm Everforest surfaces, a single
Cobalt 2 chrome identity (gold at night, deep navy by day), and
Flexoki's calm inky syntax palette.

Forest Flower ships two themes: `night` (dark) and `day` (light). By
default it follows |'background'|. See DESIGN.md for the design thesis.

FEATURES                           *forestflower.nvim-forest-flower-features*

- 100% Lua, supports Treesitter & LSP
- Vim terminal colours
- Lualine theme
- Single-file theme model: drop a Lua file to add a theme.
- Strict validation: missing palette roles fail loudly at startup, not at
  first paint.


INSTALLATION                   *forestflower.nvim-forest-flower-installation*

Using lazy.nvim: >lua
    {
      "YajanaRao/forestflower",
      priority = 1000,
      lazy = false,
      config = function()
        vim.cmd.colorscheme("forestflower")
      end,
    }
<


CONFIGURATION                 *forestflower.nvim-forest-flower-configuration*

Defaults: >lua
    require("forestflower").setup({
      -- theme = "night",                 -- omit to auto-detect via vim.o.background
      transparent_background = false,
      italics = false,
      sign_column_background = "grey",    -- "none" | "grey"
      diagnostic_text_highlight = false,
      diagnostic_virtual_text = "coloured", -- "coloured" | "grey"
      float_style = "bright",             -- "bright" | "dim"
      on_highlights = function(highlights, palette) end,
      colours_override = function(palette) end,
    })
<


RECOMMENDED `guicursor`

Neovim cannot set cursor pixel width from a colorscheme. To approximate the
spec's 5px fat cursor in a 14px font, use |'guicursor'| in your init: >vim
    set guicursor=n-v-c:block-Cursor,i-ci-ve:ver60-Cursor,r-cr:hor20-Cursor,o:hor50-Cursor,a:Cursor/lCursor,sm:block-blinkwait175-blinkoff150-blinkon175
<
`ver60` renders a bar at ~60% of cell width (~5px at 14px cell size), closer
to the DESIGN.md cursor spec than the default thin bar.

RECOMMENDED `lualine`

Forest Flower ships a native lualine theme matching DESIGN.md chrome rules:
normal mode uses `{colors.success}` (RCB green) to signal resting state;
all active-mutation modes (insert, visual, replace, command) use
`{colors.primary}` (gold/navy). Load it with: >lua
    require("lualine").setup({
      options = { theme = "forestflower" }
    })
<
If lualine is loaded before the colorscheme, ensure `forestflower` is
available on |runtimepath| or lazy-load lualine after the colorscheme.


OVERRIDES                         *forestflower.nvim-forest-flower-overrides*

Both override hooks operate on a per-load copy of the palette; mutations
never leak across loads.

Highlight overrides: >lua
    require("forestflower").setup({
      on_highlights = function(hl, palette)
        hl.DiagnosticError = { fg = palette.error }
      end,
    })
<

Palette overrides: >lua
    require("forestflower").setup({
      colours_override = function(palette)
        palette.primary = "#b86466"
      end,
    })
<


ADDING A THEME             *forestflower.nvim-forest-flower-adding-a-theme*

Drop a file under `lua/forestflower/themes/`. Every theme returns a flat
palette table; the discovery layer auto-loads any file in this directory.

The validator requires every key in the schema. Missing values fail at
load time with a clear error message.

See `lua/forestflower/themes/night.lua` for the reference implementation
and `lua/forestflower/themes/init.lua` for the required-role list.

Select a theme: >lua
    require("forestflower").setup({ theme = "my-theme" })
<
